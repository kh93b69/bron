# PROJECT_IDEA — CyberBook

> Lightweight SaaS для мгновенного бронирования ПК в кибер-аренах. Превращаем «переписку с админом в WhatsApp» в 30-секундный флоу в браузере.

---

## 1. Проблема

Конкретная боль геймера и владельца клуба, измеренная в цифрах:

| Фактор | Текущая реальность | Цена для бизнеса |
|---|---|---|
| Время ответа админа | 10–15 минут в WhatsApp/Instagram, 2–4 минуты по телефону | Геймер уходит к конкуренту за 5 минут ожидания |
| Потерянные брони | ~15–20% сообщений админ не видит (смена, шум, наушники) | Прямая недополученная выручка |
| Ошибки человеческого фактора | Двойные брони, забытые записи в тетрадке/Excel | Конфликт с клиентом на рецепции, возврат денег |
| Групповая бронь (5+ ПК) | 20–40 минут переписки, чтобы собрать команду на соседние места | Команды уходят тренироваться в другой клуб |
| Ноу-шоу | 10–25% забронировавших не приходят, админ ждёт впустую | Простой оплаченного железа |
| Непрозрачность загрузки | Клиент не видит свободных мест, едет «наудачу» | Негативный опыт, потеря лояльности |

Итог: владелец клуба теряет 15–30% потенциальной выручки в будни из-за трения на этапе «геймер → кресло».

## 2. Решение

Lightweight SaaS — не замена биллинг-системам (Langame/SmartShell/Senet), а **надстройка-витрина** поверх них.

### Пользовательский флоу (B2C, ~30 секунд)
1. Геймер кликает ссылку в bio Instagram клуба → открывается PWA
2. Видит интерактивную карту зала: зелёные места свободны, красные заняты
3. Выбирает зону (VIP / Bootcamp / General) и конкретный ПК
4. Выбирает дату и слот (1 ч / 3 ч / 5 ч / ночь)
5. Вводит имя и телефон (OTP-верификация) → получает бронь
6. PUSH-уведомление за 30 минут до начала. QR-код для ресепшена

### Флоу владельца (B2B)
1. Регистрирует клуб, получает дашборд
2. Рисует карту зала (встроенный редактор: drag-n-drop ПК по сетке)
3. Настраивает зоны, тарифы, правила
4. Вешает ссылку/QR-код в Instagram bio, на входе, в Telegram-канале
5. Админ на ресепшене видит панель с live-бронями + звуковое уведомление
6. В конце смены — дашборд: загрузка по часам, популярные ПК, revenue

### Что это даёт бизнесу
- Снижение нагрузки на админа на 60–80% (нет переписок)
- +15–25% броней в будни за счёт «горящих мест»
- База клиентов с телефонами → ретаргетинг, SMS-рассылки, программа лояльности
- Аналитика по реальной загрузке каждого ПК

## 3. Почему сейчас

- **Рынок готов**: после 2020 года геймеры привыкли к бронированию через QR (рестораны, салоны, коворкинги). Барьер «я не умею онлайн» исчез.
- **Instagram-first поколение**: 70% клиентов клубов (14–25 лет) приходят через соцсети. Ссылка в bio — естественный канал.
- **Конкуренты-тяжеловесы не двигаются**: Langame/SmartShell/Senet — это биллинговые системы с 10-летним легаси, их модули бронирования требуют установки приложения и неудобны.
- **PWA-технологии зрелые**: можно выдать нативный UX без App Store/Google Play — критично для КЗ/СНГ, где клиенты не хотят ставить отдельные приложения.
- **Географическое окно**: в Казахстане (Алматы, Астана, Караганда, Шымкент) — 400+ клубов, только единицы имеют онлайн-бронирование. Первый игрок заберёт рынок.

## 4. Целевая аудитория

### Основная (B2B — плательщик)
**Владельцы и управляющие компьютерных клубов**
- Сегменты: независимые клубы (20–60 ПК, 1 локация), локальные сети (2–5 локаций), франшизы (Colizeum, CyberX, ColizeumKZ)
- Боль: текучка админов, потерянные брони, невозможность масштабировать ручной процесс
- Текущие инструменты: тетрадка, Excel, Google Sheets, встроенный модуль биллинга
- Готовность платить: 10 000–40 000 ₸/мес за клуб, если система приносит +1 бронь/день

### Вторичная (B2C — пользователь)
**Геймеры 14–30 лет**
- Ценят скорость и отсутствие социального взаимодействия (не хотят звонить/писать)
- Играют 2–10 раз в месяц, средний чек 1500–5000 ₸ за сессию
- Текущий флоу: WhatsApp клуба, Instagram Direct, звонок

**Киберспортивные команды (5 человек)**
- Нужна уверенность в 5 соседних ПК для тренировок (пракков)
- Бронируют за 1–7 дней вперёд
- Готовы платить депозит за гарантию мест

## 5. Архитектура (логическая, без привязки к стеку)

### Слои системы

```
┌─────────────────────────────────────────────────────┐
│ L1. Клиентский PWA — витрина + бронирование          │
│     (Instagram link → карта → бронь → QR)           │
├─────────────────────────────────────────────────────┤
│ L2. Админ-панель клуба                              │
│     (дашборд, live-брони, редактор карты, черный    │
│      список, аналитика)                             │
├─────────────────────────────────────────────────────┤
│ L3. Super-admin (SaaS-owner)                        │
│     (управление клубами, подписками, биллингом)     │
├─────────────────────────────────────────────────────┤
│ L4. API / серверная логика                          │
│     (бронирования, уведомления, платежи, RLS)       │
├─────────────────────────────────────────────────────┤
│ L5. База данных (PostgreSQL + RLS)                  │
├─────────────────────────────────────────────────────┤
│ L6. Интеграции (уведомления, платежи, SMS OTP)      │
└─────────────────────────────────────────────────────┘
```

### Технический стек (зафиксирован)

- **Хостинг**: Railway (Node runtime + managed integrations)
- **VCS / CI**: GitHub → Railway auto-deploy на push в `main`
- **Backend + БД + Auth + Realtime**: Supabase (Postgres 15, RLS, встроенный Auth с Email OTP)
- **Фреймворк**: Next.js 15 (App Router, Route Handlers, Server Actions) + React 19
- **Язык**: TypeScript strict
- **UI**: Tailwind CSS v4 + shadcn/ui + Lucide
- **Данные**: `@supabase/ssr` + TanStack Query v5
- **Формы**: React Hook Form + Zod
- **PWA**: стандартные next/pwa-плагины (manifest + service worker)
- **Email**: Resend (транзакционные письма) + Supabase Auth (OTP/magic link)

**Не в MVP**: платежи (Kaspi/Freedom/Stripe), SMS, WhatsApp Business API, мобильное приложение, кастомные домены white-label. Всё это — Фаза 4+.

### Модули

| Модуль | Вход | Выход |
|---|---|---|
| `auth` | email/телефон + OTP | session, роль (player/club_admin/super_admin) |
| `clubs` | owner_id, данные клуба | CRUD клубов |
| `zones` | club_id, тип (VIP/Bootcamp/General), тариф/час | зоны с ПК |
| `stations` | zone_id, позиция (x,y), спецификация ПК | места на карте |
| `map_editor` | club_id, layout JSON | сохранённая карта зала |
| `bookings` | user_id, station_id, slot | бронь + уведомление админу |
| `group_bookings` | user_id, station_ids[], slot | атомарная бронь соседних ПК |
| `notifications` | event (bookingCreated/cancelled/reminder) | push/Telegram/SMS |
| `reputation` | user_id, history | рейтинг, автобан после 2 no-show |
| `blacklist` | club_id, user_id, reason | запрет на брони в клубе |
| `promotions` | club_id, правила (время/скидка) | «Горящие места» с ценой |
| `payments` | user_id, depositAmount | депозит через Kaspi/Freedom (Pro) |
| `subscriptions` | club_id, plan | биллинг B2B-клубов |
| `analytics` | club_id, период | дашборд: загрузка, revenue, топ-ПК |

Гейтинг:
- Free → Start: безлимит броней, экспорт, веб-админка

## 7. Конкуренты

| Конкурент | Что делает | Чего не хватает | Наш ответ |
|---|---|---|---|
| **Langame** | Биллинг + управление ПК + модуль броней | Бронирование требует отдельного приложения, сложный UX, дорого | PWA без установки, в 3× дешевле, стыкуется поверх |
| **SmartShell** | Биллинг, сессии, оплата | Нет клиентской витрины, только админка | Мы даём B2C-интерфейс |
| **Senet** | Биллинг, контент-контроль | Модуль броней устарел, нет мобильной версии | Mobile-first PWA |
| **Google Forms + Excel** | «Самодельное» бронирование | Нет live-карты, ручной процесс | Автоматизация + визуализация зала |
| **WhatsApp/Instagram** | Прямое общение с админом | Медленно, ошибки, нет памяти системы | 30-секундный флоу + база клиентов |
| **Booksy / YClients** | Универсальный сервис записи (салоны, фитнес) | Нет концепции «карта зала с ПК», не заточено под игровой контекст | Вертикальная специализация |

Ключевая дифференциация: **мы не конкурируем с биллингом, мы живём поверх него.** Клубу не нужно менять Langame — он просто добавляет CyberBook как «онлайн-витрину».

## 8. План запуска (переделанный роадмап)

> Исходный роадмап объединял прототип и fake-it MVP, но рационально разделить по фазам с чёткой метрикой валидации на каждой.

### Фаза 0 — Дизайн-валидация (1 неделя)
**Цель**: получить «вау»-эффект у владельцев клубов до написания бэкенда.
- High-Fi кликабельный прототип в Figma (или сразу на фронте): карта зала, выбор ПК, подтверждение, админка с графиками
- Демо-версия живёт по публичной ссылке
- Провести 5–10 кастдев-интервью с владельцами клубов (Алматы, Караганда)
- **Метрика выхода**: ≥3 владельца сказали «когда запустите — возьму»

### Фаза 1 — Концьерж-MVP (2 недели)
**Цель**: валидировать спрос со стороны геймеров без полноценного бэкенда.
- Публичная PWA-витрина одного клуба (реальный тестовый партнёр)
- Кнопка «Забронировать» отправляет заявку в Telegram-бота → админ подтверждает вручную
- Минимальная модель данных (брони + клиенты), без карты-редактора, без платежей
- **Метрика выхода**: ≥50 броней через виджет за 2 недели, CR из клика по ссылке в bio ≥15%

### Фаза 2 — Полный MVP (4 недели)
**Цель**: система работает автономно, админ не обрабатывает брони вручную.
- Полный стек: БД + RLS, админ-панель клуба, редактор карты, live-обновления статуса ПК
- Уведомления: в браузер админа (Web Push + звук) и клиенту (SMS OTP)
- Роли: player, club_admin
- Рейтинг клиентов, автобан после 2 no-show
- **Метрика выхода**: пилотный клуб работает 2 недели без ручной обработки броней

### Фаза 3 — Пилот (2 недели)
**Цель**: получить кейс + первые деньги.
- Бесплатный Pro-тариф на 2 недели для 2–3 клубов в Алматы/Караганде
- Ежедневный фидбэк от админов и геймеров
- Сравнение недели «до» и «после» по количеству броней
- **Метрика выхода**: ≥1 клуб согласился на платную подписку, +15% броней в будни

### Фаза 4 — Платный запуск (1 месяц)
**Цель**: первая выручка.
- Ручной биллинг подписок: счёт-фактура по Kaspi Business / банковский перевод (без автосписания на старте — 5 клиентов можно провести руками)
- Super-admin панель: список клубов, продление/смена тарифа, отключение неплатящих
- Продающий лендинг + первый кейс пилотного клуба
- **Метрика выхода**: 5 платящих клубов, MRR ≥ 100 000 ₸

### Фаза 4.5 — Платежи под ключ (опционально, через 2–3 месяца после Фазы 4)
**Цель**: автоматизировать биллинг и ввести депозиты для борьбы с no-show.
- Интеграция Kaspi Pay / Freedom — B2B-подписки с автосписанием и B2C-депозиты
- Возвраты при check-in, списание при no-show
- Включение гейтинга фич Pro / Network через `hasFeature()`

### Фаза 5 — Scale (постоянно)
- «Горящие места» (dynamic pricing для простоя утром)
- Групповые брони (5 ПК подряд) с депозитом
- Аналитика для сетей (cross-club dashboard)
- White-label для франшиз
- Мобильное приложение (если PWA окажется недостаточно)
- Программа лояльности, промокоды, реферальная программа

### Разделение MVP / v2

**MVP (Фазы 1–3)**: auth (Email OTP через Supabase), clubs, zones, stations, bookings, notifications (email + web push), reputation, club-admin-panel, map-editor
**v2 (Фазы 4+)**: SMS-напоминания, blacklist, promotions (горящие места), group_bookings UX, analytics-dashboard, super-admin full
**v3 (Фаза 4.5+)**: payments (Kaspi/Freedom депозиты + автосписание подписок), white-label, API-доступ

## 9. Риски и митигация

| Риск | Вероятность | Митигация |
|---|---|---|
| No-show: клиенты бронируют и не приходят | Высокая (10–25%) | **MVP**: рейтинг клиента, автобан 7 дней после 2 no-show, явное предупреждение при бронировании. **v3 (после фазы 4.5)**: депозит 500 ₸ через Kaspi (возвращается при check-in) |
| Конфликт с биллингом клуба: админ забывает отметить бронь | Средняя | Планшет на ресепшене с всегда-открытой админ-панелью + громкий звук новой брони. Фаза 4: интеграция-webhook в Langame/SmartShell |
| Клуб не хочет рисовать карту зала | Высокая | Услуга «Отрисовка под ключ» — бесплатно для годовых Pro/Network. Минимальный шаблон: сетка 8×6 без визуала |
| Геймеры не знают про систему, привычка WhatsApp | Высокая | Печатные QR-коды на входе/барной стойке + промо-пост в инсте клуба «бронируй онлайн — первый час –20%» |
| Двойная бронь (race condition) | Средняя | Транзакционная блокировка station_id на уровне БД + оптимистичная UI-проверка |
| Падение сервера в пятницу вечером | Низкая, но критично | SLA 99.5%, graceful-degradation: если бэк недоступен — PWA показывает «забронируй по WhatsApp», автоматический фолбэк |
| Конкурент-тяжеловес (Langame) выкатывает PWA | Средняя (1–2 года) | Успеть занять долю рынка за 12 месяцев, сфокусироваться на UX и скорости |
| Регуляторные ограничения для несовершеннолетних (КЗ) | Низкая | Ввод возраста при регистрации, фильтр броней после 22:00 для <18 (требование закона) |
| Кража базы телефонов | Средняя по impact | RLS + шифрование, логирование доступа, 2FA для super-admin |

### Ключевые таблицы (предварительный список, детали — в SPEC.md)
- `users` — игроки и админы (unified)
- `clubs` — локации клубов
- `club_members` — кто кем управляет (owner/admin)
- `zones` — VIP/Bootcamp/General с ценами
- `stations` — ПК с координатами на карте
- `bookings` — сами брони
- `booking_stations` — связь бронь↔ПК (для групповых)
- `user_reputation` — рейтинг и история no-show
- `blacklists` — бан по клубам
- `promotions` — горящие места, правила скидок
- `subscriptions` — B2B-биллинг
- `payment_intents` — депозиты игроков
- `notifications_log` — история отправленных уведомлений

### Критические non-functional требования
- **Time-to-book**: ≤30 секунд от клика по ссылке до подтверждения (mobile 4G)
- **Page load**: ≤2 секунды до интерактивной карты (LCP)
- **Concurrent bookings**: корректная обработка 50+ одновременных попыток на 1 ПК
- **Uptime**: 99.5% в часы работы клубов (12:00–04:00 GMT+5)
- **Offline-tolerance**: PWA показывает последнее известное состояние при потере сети
- **Языки**: RU, KZ, EN (на интерфейс)
- **Валюта**: ₸ (MVP), далее ₽, ₴, Сом

### Логика no-show и рейтинга
```
user_reputation.score:
  start = 100
  -30 за no-show
  -15 за отмену <2 ч до начала
  +5 за каждую успешную бронь
  score < 40 → автоматический бан на 7 дней в clubs, где был no-show
```

### AI-пайплайн
В MVP отсутствует. В v2 рассматривается: рекомендация «лучшего» ПК пользователю по истории, генерация промо-текстов для клубов. Не приоритет.
